수업을 듣는 중, 배열을 복사하는 메소드가 여러 개 있는 것을 알게 되었다. `System.arraycopy()`와 `Arrays.copyOf()`, `clone()`이 존재한다. 세 메소드는 모두 배열을 복사한다는 공통점이 있지만, 사용 목적과 특성에는 차이가 있다. 

### `System.arraycopy()`

```java
System.arraycopy(원본배열, 원본 배열의 복사 시작 지점, 복사할 배열, 
								복사할 배열의 복사 시작 지점, 복사할 요소의 개수)
```

복사할 배열의 시작 지점과 개수를 설정할 수 있기 때문에, 특정 부분의 데이터를 복사하기에 장점이 있다. 

### `Arrays.copyOf()`

```java
Arrays.copyOf(원본배열, 원본 배열에서 복사해올 길이)
```

`System.arraycopy()`와 다르게 시작 인덱스가 `0`으로 고정되어 있어, 특정 부분 데이터를 복사할 수는 없다. 

### `clone()`

```java
int[] a = {1, 2, 3};
int[] b = a.clone();
```

배열 전체를 복사하는 가장 간단한 방법이다. 새로운 배열 객체가 생성되며, 배열의 길이를 조정하거나 일부분만 복사하는 것은 불가능하다. 

## 그럼 어떤걸 써야해요?

`System.arraycopy()`의 경우 네이티브 메소드(C나 C++같은 네이티브 프로그래밍 언어로 작성한 메서드) 이다. 배열의 크기가 큰 경우 반복문보다 배열 복사를 빠르게 가능하다. 또한 복사할 배열과 복사 되는 배열이 같아도 안전하게 동작한다.

`Arrays.copyOf()`의 경우 `System.arraycopy()`에 기능을 추가한 메소드이므로 일반적인 경우 성능 차이는 거의 체감되지 않는다. 

`clone()`의 경우 가장 직관적인 사용이지만 배열의 크기를 조정할 수 없다는 점이 단점이다. 가장 간단하게 쓸 때 좋을 것 같다. 

## 2차원 배열 복사 시 주의할 점

2차원 배열을 복사할 경우, 세 메소드 모두 얕은 복사가 이루어진다. 바깥 배열은 새로 생성되지만, 내부 배열은 기존 배열과 동일한 객체를 참조한다. 

따라서 복사된 배열에서 내부 배열의 값을 반환하면, 원본 배열의 값도 함께 변경되는 문제가 발생할 수 있다.
