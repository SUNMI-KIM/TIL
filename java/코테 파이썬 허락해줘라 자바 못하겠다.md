### 이클립스 단축키

아래 내가 자주 쓰는 이클립스 단축키를 정리했다. 단축키 종류는 더 많지만 내가 쓰는 것만 정리해뒀다.

- `System.out.println()` → `syso` 입력 후 `ctrl + space`
- 자동 포맷팅 → `ctrl + shift + F`
- `main method → m` 입력 후 `ctrl + space`

### 이클립스는 한번에 모든 프로젝트를 관리해?

![](https://velog.velcdn.com/images/hariaus/post/4c830209-a6ff-4b60-bdf3-ddf06a4c95d6/image.png)


인텔리제이는 한번에 하나의 프로젝트를 관리했다면 이클립스는 여러개의 프로젝트를 한번에 관리 가능하다.

인텔리제이는 기본적으로 하나의 프로젝트 단위로 작업하는 반면, 이클립스는 하나의 워크스페이스 안에서 여러 프로젝트를 동시에 관리 가능하다. 

### 근데 프로젝트에 어떻게 들어가지?

![](https://velog.velcdn.com/images/hariaus/post/ba5c127c-2ff6-4cc0-a36f-af336a5deed4/image.png)


프로젝트를 생성하고 Open 했는데 프로젝트가 화면에 보이지 않았다. 프로젝트를 보려면 Welcome 옆에 있는 작은 윈도우 창을 누르면 된다.

### 콘솔 입력 받기 (Scanner)

자바로 코딩 테스트를 거의 해본 적이 없어, 처음에는 콘솔 입력을 어떻게 받아야 하는지도 몰랐다. 가장 기본적은 방식은 `Scanner`를 사용하는 방법이 있다. 

```jsx
Scanner scanner = new Scanner(System.in);
String input = scanner.nextLine();
```

### 더 빠르게 입력 받기

입력 데이터의 양이 많을 경우, Scanner는 성능이 느려 시간 초과가 발생할 수 있다. 이런 경우에는 `BufferedReader`를 사용하는 것이 좋다.

`BufferedReader`는 입력 데이터를 버퍼에 모아 처리하기 때문에 성능이 더 빠르다. 

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
int test_case = Integer.parseInt(br.readLine());
br.close(); // 사용한 자원 해제
```

입력된 문자열을 분리할 때는 `split()`또는 `StringTokenizer`를 사용할 수 있다. 

```java
String[] input = br.readLine().split(" ");

StringTokenizer st = new StringTokenizer(br.readline());
```

`split()`의 경우 데이터가 많을 때, `StringTokenizer`보다 느릴 수 있다. 

StringTokenizer는 레거시 클래스라 실무 프로젝트에선 지양하는 편이지만, 코딩 테스트 환경에서는 성능 측면에서 사용해도 무방하다고 판단했다. 

### 더 빠르게 출력하기

`System.out.println();`은 출력이 많을 경우 성능 저하가 발생할 수 있다. 이럴 때는 `StringBuilder` 또는 `BufferedWriter`를 사용하면 좋다. 

```java
StringBuilder sb = new StringBuilder();
sb.append("Hello").append(",");
System.out.println(sb);

BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
bw.write(); // 출력 모아두기
bw.newLine();
bw.flush(); // 한번에 출력하기
```

주변 사람들의 이야기를 들어보니, 대부분 출력이 많은 경우 `StringBuilder`를 가장 많이 사용하는 것 같다. 

- 출력이 적을 때 → `System.out.println()`
- 출력이 많을 때 → `Stringbuilder`(또는 `BufferedWriter`)
