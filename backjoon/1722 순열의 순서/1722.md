## Next Permutation


알고리즘 문제를 풀다 보면, 순열의 개수를 구하거나 최댓값을 구하기 위해 모든 순열을 탐색해야 하는 경우가 있다. 

하지만 다음 순열(next permutation)을 구하는 문제에서는 모든 순열을 직접 생성하는 방식은 비효율적이며, 특히 `n = 20`과 같이 경우의 수가 매우 큰 경우에서는 시간 초과가 발생한다. 

따라서 재귀나 완전 탐색 대신, 순열의 규칙성을 이용한 알고리즘을 사용해야 한다. 

### Next Permutation 알고리즘

1. 배열의 가장 오른쪽에서 시작해 왼쪽으로 이동하며, 처음으로 값이 감소하는 지점의 인덱스(Pivot)를 찾는다. 
    
![](https://velog.velcdn.com/images/hariaus/post/e79fb38a-dc32-4b48-957b-8cd1863cb93d/image.jpg)

    
2. 배열의 가장 오른쪽부터 다시 탐색하여, Pivot보다 크면서 가장 작은 값의 인덱스를 찾는다. 
    
![](https://velog.velcdn.com/images/hariaus/post/fa72334d-1851-4113-8e0e-f01590d55079/image.jpg)

    
3. Pivot과 해당 값을 swap 한다. 
    
![](https://velog.velcdn.com/images/hariaus/post/9b201c96-6b40-4605-8eb1-9a69b8233988/image.jpg)

    
4. Pivot의 오른쪽 부분을 오름차순으로 만든다. 

![](https://velog.velcdn.com/images/hariaus/post/cbd57e34-b9b7-435a-a1ca-edf67f95dcf9/image.jpg)



    

## 시간 초과 코드 (Java)

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class _1722 {
	
	static int n;

	public static void main(String[] args) throws Exception {
		// TODO Auto-generated method stub
		BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
		StringBuilder sb = new StringBuilder();
		
		n = Integer.parseInt(br.readLine());
		
		StringTokenizer st = new StringTokenizer(br.readLine());
		int flag = Integer.parseInt(st.nextToken());
		if (flag == 1) {
			int[] arr = new int[n];
			for (int i = 0; i < n; i++) {
				arr[i] = i + 1;
			}
			
			one(arr, Integer.parseInt(st.nextToken()));
			for (int i = 0; i < n; i++) {
				sb.append(arr[i]).append(" ");
			}
		} else {
			int[] arr = new int[n];
			for (int i = 0; i < n; i++) {
				arr[i] = Integer.parseInt(st.nextToken());
			}
			int count = two(arr);
			sb.append(count);
		}
		System.out.println(sb.toString());
	}
	
	static void one(int[] arr, int count) {
		
		for (int i = 0; i < count - 1; i++) {
			int pivot = findPivot(arr);
			if (pivot == -1) break;
			
			int swapIdx = findSwapIndex(arr, pivot);
			
			swap(arr, pivot, swapIdx);
			Arrays.sort(arr, pivot + 1, n);
		}
	}
	
	static int two(int[] arr1) {
		int ans = 0;
		int[] arr2 = new int[n];
		
		for (int i = 0; i < n; i++) {
			arr2[i] = i + 1;
		}
		
		while (!equals(arr1, arr2)) {
			int pivot = findPivot(arr2);
			if (pivot == -1) break;
			
			int swapIdx = findSwapIndex(arr2, pivot);
			
			swap(arr2, pivot, swapIdx);
			Arrays.sort(arr2, pivot + 1, n);
			ans++;
		}
		return ans + 1;
	}
	
	static boolean equals(int[] arr1, int[] arr2) {
		for (int i = 0; i < n; i++) {
			if (arr1[i] != arr2[i]) return false;
		}
		return true;
	}
	
	static void swap(int[] arr, int left, int right) {
		int tmp = arr[right];
		arr[right] = arr[left];
		arr[left] = tmp;
	}
	
	static int findPivot(int[] arr) {
	    for (int i = arr.length - 1; i > 0; i--) {
	        if (arr[i] > arr[i - 1]) {
	            return i - 1;
	        }
	    }
	    return -1;
	}
	
	static int findSwapIndex(int[] arr, int pivot) {
	    int min = Integer.MAX_VALUE;
	    int idx = -1;

	    for (int i = pivot + 1; i < arr.length; i++) {
	        if (arr[i] > arr[pivot] && arr[i] < min) {
	            min = arr[i];
	            idx = i;
	        }
	    }
	    return idx;
	}

}

```

## 문제 풀이 방식


처음에는 Next Permutation을 직접 구현하여 문제를 풀려고 시도했다. 하지만 이 문제는 n ≤ 20 이고, `20! = 2,432,902,008,176,640,000` 으로 경우의 수가 2경에 달한다. 

따라서 Next Permutation을 반복 실행하는 방식은 시간 제한 내에 절대 해결할 수 없다. 결국 순열의 규칙성을 수학적으로 분석하여 해결해야 한다. 

### 입력 값을 나열해보자

```java
1 2 3 4  (1)
1 2 4 3  (2)
1 3 2 4  (3)
1 3 4 2  (4)
1 4 2 3  (5)
1 4 3 2  (6)

2 1 3 4  (7)
2 1 4 3  (8)
2 3 1 4  (9)
2 3 4 1  (10)
2 4 1 3  (11)
2 4 3 1  (12)

3 1 2 4  (13)
3 1 4 2  (14)
3 2 1 4  (15) 
3 2 4 1  (16)
```

### 앞자리에 기준으로 보이는 규칙

- `1 _ _ _` → `3! = 6`개
- `2 _ _ _` → `3! = 6`개

즉, `3`으로 시작하는 순열은 12 + 1 = 13번째 부터 시작한다. 

### 두 번째 자리도 같은 규칙이 적용된다.

`3`으로 시작하는 경우를 보면,

- `3 1 _ _` → `2! = 2`개
- `3 2 _ _` → `2! = 2`개

따라서 `3 4 _ _`는 17번째 순열부터 시작한다. 

### 여기서 발견한 규칙

1. 현재 남아있는 숫자의 개수를 `k` 라고 할 때, 한 숫자가 고정되면 `(k - 1)!` 개의 순열이 묶인다. 
2. `(k - 1)!`을 기준으로 몇 번째 묶음에 속하는지 계산한다. 
3. 이미 사용한 숫자는 제외해야 하므로, 상대 인덱스 개념이 필요하다. 
4. 이 과정을 모든 자리에서 반복한다. 

### 상대 인덱스의 필요성

예를 들어 `N = 4`, `16번째 순열`을 구한다고 하자.

- 처음에 `3! = 6`을 두 번 빼면,
    
    첫 번째 숫자는 **3**
    
- 남은 `count = 3`
- 다음으로 `2! = 2`를 한 번 빼면,
    
    두 번째 숫자는 **2**
    

하지만 다음 단계에서 `count = 1`이고 `1! ≠ 1` 이므로, 이미 사용한 값(2)를 다시 선택하면 안된다.   

즉, 여기서 사용하는 인덱스는 절대 인덱스가 아니라 “**아직 사용하지 않은 값들 중에서의 상대 인덱스**”이다. 

### visited 배열을 사용하는 이유

배열을 계속 복사하는 방식도 가능하지만, 비효율적이라고 판단하여 `visited` 배열을 사용했다. 

```java
for (int num = 0; num < n; num++) {
				if (visited[num]) continue;
				
				if (idx-- == 0) {
					visited[num] = true;
					arr[i] = num + 1;
					break;
				}
			}
```

이 방식으로 남아 있는 값 중 `idx`번째를 정확히 선택할 수 있다. 

### flag = 2 (순열 → 몇 번째인지)

`flag`가 `2`인 경우는 위 과정을 역으로 적용하면 된다. 

예를 들어 `N = 4`, 순열이 `3 2 4 1`이라면,

- 첫 번째 자리 `3`
    
    → 앞에 올 수 있었던 값 `1, 2` → `2 × 3!`
    
- 두 번째 자리 `2`
    
    → 앞에 올 수 있었던 값 `1` → `1 × 2!`
    
- 세 번째 자리 `4`
    
    → 앞에 올 수 있었던 값 `1` → `1 × 1!`
    
- 마지막 자리는 계산할 필요 없음

즉, **(현재 값보다 작은 남아 있는 값의 개수) × (남은 자리 수)!** 를 누적해서 더하면 된다.

## 최종 코드 (Java)

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class _1722 {

	static int n;
	static long[] factorial = new long[21];
	static int[] arr;
	static boolean[] visited;

	public static void main(String[] args) throws Exception {
		// TODO Auto-generated method stub
		init();
		BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
		StringBuilder sb = new StringBuilder();

		n = Integer.parseInt(br.readLine());
		arr = new int[n];
		visited = new boolean[n];

		StringTokenizer st = new StringTokenizer(br.readLine());
		int flag = Integer.parseInt(st.nextToken());
		if (flag == 1) {
			one(Long.parseLong(st.nextToken()) - 1);
			for (int i = 0; i < n; i++) {
				sb.append(arr[i]).append(" ");
			}
		} else {
			for (int i = 0; i < n; i++) {
				arr[i] = Integer.parseInt(st.nextToken());
			}
			sb.append(two());			
		}
		System.out.println(sb.toString());
	}
	

	static void init() {
		factorial[0] = 1;
		for (int i = 1; i < 21; i++) {
	        factorial[i] = factorial[i - 1] * i;
	    }
	}
	
	static void one(long count) {
		for (int i = 0; i < n; i++) {
			int idx = (int) (count / factorial[n - i - 1]);
			
			for (int num = 0; num < n; num++) {
				if (visited[num]) continue;
				
				if (idx-- == 0) {
					visited[num] = true;
					arr[i] = num + 1;
					break;
				}
			}
			count %= factorial[n - i - 1];
		}
	}
	
	static long two() {
		long ans = 0;
		for (int i = 0; i < n - 1; i++) {
			int count = 0;
			for (int r = arr[i] - 2; r >= 0; r--) {
				if (visited[r]) continue;
				count++;
			}
			ans += count * factorial[n - i - 1];
			visited[arr[i] - 1] = true;
		}
		return ans + 1;
	}

}
```
