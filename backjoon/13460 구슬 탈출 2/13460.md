## 문제 풀이 방식

---

이 문제는 특정 복잡한 알고리즘을 요구한다기보다, 구현(시뮬레이션)과 완전 탐색(DFS/BFS) 설계를 얼마나 꼼꼼하게 하느냐가 관건인 문제였다. 

구슬이 두 개라 굴러가다가 서로 부딪히는 경우나 겹치는 경우를 예외 처리해야 해서 상당히 까다로웠다. 나는 DFS(백트래킹)를 통해 문제를 풀었지만, 최소 이동 횟수를 구하는 문제의 특성 상 처음부터 최단 거리를 보장하는 BFS를 사용하는 것이 더 직관적이고 빠를 수 있겠다. 

### 1. 입력을 받고 각 구슬의 위치를 찾기

```java
for (int i = 0; i < n; i++) {
		arr[i] = br.readLine().toCharArray();
		for (int r = 0; r < m; r++) {

				red = new Pair(i, r);
				arr[i][r] = '.';
			}
		}
	}
```

입력 배열을 순회하며 빨간 구슬과 파란 구슬의 위치를 찾아 객체에 저장한다. 이후 구슬이 굴러 다니는 데 방해되지 않도록 해당 자리는 빈칸으로 바꾼다.

### 2. 백트래킹을 통해 각 구슬의 위치를 이동시킨다.

```java
static void solve(int depth, Pair red, Pair blue) {
		if (depth > 10)
			return;

		for (int i = 0; i < 4; i++) {
			Pair preRed = new Pair(red.x, red.y);
			Pair preBlue = new Pair(blue.x, blue.y);

			// 여기에 판별 로직 적용
			boolean status = lean(i, preRed, preBlue);
			if (status) {
				if (arr[preBlue.x][preBlue.y] == 'O') continue;
				if (arr[preRed.x][preRed.y] == 'O') {
					min = Math.min(min, depth);
					continue;
				}
			}
			solve(depth + 1, preRed, preBlue);
		}
	}

```

백트래킹을 통해 구슬들을 상하좌우로 굴려본다. 이때 파란 구슬이 구멍에 들어간 경우는 무조건 실패이므로 다음 탐색(다른 방향)으로 넘어가고, 빨간 구슬만 들어갔다면 최소 횟수를 갱신한다. 원본 좌표가 훼손되지 않도록 `preRed`, `preBlue`로 복사본을 만들어 넘기는 것이 핵심이다.

### 3. 구슬을 상하좌우 방향으로 이동시킨다.

```java
static boolean lean(int i, Pair red, Pair blue) {
		boolean status = false;

		int red_count = 0;
		while (true) {
			int nx = red.x + dx[i];
			int ny = red.y + dy[i];
			if (nx < 0 || nx >= n || ny < 0 || ny >= m)
				break;
			if (arr[nx][ny] == '#')
				break;
			if (arr[nx][ny] == 'O') {
				status = true;
				red.x = nx;
				red.y = ny;
				break;
			}
			red_count++;
			red.x = nx;
			red.y = ny;
		}

		int blue_count = 0;
		while (true) {
			int nx = blue.x + dx[i];
			int ny = blue.y + dy[i];
			if (nx < 0 || nx >= n || ny < 0 || ny >= m)
				break;
			if (arr[nx][ny] == '#')
				break;
			if (arr[nx][ny] == 'O') {
				status = true;
				blue.x = nx;
				blue.y = ny;
				break;
			}
			blue_count++;
			blue.x = nx;
			blue.y = ny;
		}

		if (red.x == blue.x && red.y == blue.y) {
		    if (arr[red.x][red.y] != 'O') {
		        if (red_count > blue_count) {
		            red.x -= dx[i]; 
		            red.y -= dy[i];
		        } else {
		            blue.x -= dx[i];
		            blue.y -= dy[i];
		        }
		    }
		}
		return status;
	}
}

```

처음엔 Queue에 두 구슬을 담아 한 칸씩 같이 이동시키려고 했으나, 벽이나 다른 구슬에 막혀 무한 루프에 빠지는 현상이 발생하여 방식을 바꾸었다.

변경된 로직은 서로의 위치를 신경쓰지 않고 일단 각각 벽이나 구멍을 만날 때까지 끝까지 굴리는 것이다. 

1. 각 구슬을 끝까지 굴리며 이동한 칸 수(count)를 잰다.
2. 이동이 끝난 후 두 구슬의 좌표가 같고 구멍에 빠진 게 아니라면 겹친 것이다. 
3. 이때, 이동 거리가 더 길었던(원래 더 뒤에서 출발했던) 구슬을 현재 방향의 반대로 한 칸 물러나게 위치를 조정하여 겹칩 문제를 해결했다.

import java.io.InputStreamReader;

	static int m;
	static char[][] arr;

			}

		System.out.println(min);
	}

		if (depth >= min) return;

		}
	}

				status = true;

				break;

			int nx = blue.x + dx[i];
			int ny = blue.y + dy[i];
			if (nx < 0 || nx >= n || ny < 0 || ny >= m)

			if (arr[nx][ny] == '#')
				break;
			if (arr[nx][ny] == 'O') {
				status = true;
				blue.x = nx;
