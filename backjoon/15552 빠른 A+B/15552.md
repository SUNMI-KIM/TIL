## 자바로 백준 문제 풀 때 주의점


자바로 백준을 풀 때는 클래스 이름을 `Main`으로 두어야 한다. 패키지는 작성하지 않는다. 

```java
import ...

public class Main { 
	public static void main(String[] args){
    
    }
}

class 클래스명 {

}
```

## 문제 풀이 방식


문제를 읽어보면 알 수 있듯이, 빠른 입출력을 요구하는 문제이다. 자바 콘솔 입출력 내용은 아래 글에 자세히 정리해두었다.

간단히 요약하자면, `Scanner`는 느리고 `BufferedReader`는 빠르다. `System.out.println()`을 반복 호출하면 느리고, `StringBuilder`나 `BufferedWriter`로 출력을 모아서 한 번에 출력하는 방식이 빠르다. 

### `BufferedReader`와 `StringTokenizer`

빠른 입출력과 입력에 대한 파싱을 위해`BufferedReader`와 `StringTokenizer`를 사용했다.

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
int test_case = Integer.parseInt(br.readLine());

StringTokenizer st = new StringTokenizer(br.readline());
st.nextToken() // 공백을 기준으로 토큰 분리
```

### `StringTokenizer`

`System.out.println()`을 사용하되, 한번에 출력이 가능하도록 `StringTokenizer`를 사용했다. 

```java
StringTokenizer st = new StringTokenizer(br.readLine());
sb.append(A + B).append("\n");
System.out.println(sb);
```

## 최종 코드 (Java)

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {

	public static void main(String[] args) throws IOException {
		
		BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
		StringBuilder sb = new StringBuilder();
		
		int test_case = Integer.parseInt(br.readLine());
		for (int i = 0; i < test_case; i++) {
			StringTokenizer st = new StringTokenizer(br.readLine());
			int A = Integer.parseInt(st.nextToken());
			int B = Integer.parseInt(st.nextToken());
			
			sb.append(A + B).append("\n");
		}
		System.out.println(sb);
		
	}

}
