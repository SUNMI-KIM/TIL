## Tree


트리는 노드(Node)로 이루어진 자료구조이다. 

그래프의 한 종류로 볼 수 있기 때문에, 트리를 이해하기 전에 그래프를 먼저 학습하면 도움이 된다. 

1. 트리는 하나의 루트 노드를 가진다. 
2. 루트 노드는 0개 이상의 자식 노드를 가진다.
3. 각 자식 노드 또한 0개 이상의 자식 노드를 가질 수 있다.
4. 노드들은 간선(Edge)으로 연결되어 있다.
5. 사이클이 존재하지 않는다. 

## 문제 풀이 방식


이 문제는 처음에는 탐색 없이도 풀 수 있을 것이라 생각했지만, 결과적으로 탐색이 반드시 필요한 문제였다. 

### 탐색하지 않고 부모를 찾을 수 있을까?

처음에는 부모 정보를 저장하는 배열 하나만으로 문제를 해결하려고 했다.

```java
				// 부모 배열
        int[] tree = new int[n + 1];
        tree[1] = 1; // 루트 표시

        // 간선 입력 받으면서 바로 부모 결정 (❌ 핵심 오류 지점)
        for (int i = 0; i < n - 1; i++) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            int n1 = Integer.parseInt(st.nextToken());
            int n2 = Integer.parseInt(st.nextToken());

            // "이미 부모가 정해진 쪽이 부모일 것이다"라는 가정
            if (tree[n1] == 0) {
                int tmp = n1;
                n1 = n2;
                n2 = tmp;
            }

            tree[n2] = n1;
        }
```

이 방식은 부모가 둘 다 정해지지 않은 상황에서 논리적인 오류가 발생할 수 있다. 

```
5
4 5
2 4
3 2
1 3

1
└─3
  └─2
    └─4
      └─5
```

여기서 4는 5의 부모 노드이다. 하지만 `if (tree[n1] == 0)` 로직 때문에, 입력 순서에 따라 부모-자식 관계가 잘못 결정될 수 있다.

즉, 이 풀이는 입력 순서에 따라 정답이 달라지는 코드라는 치명적인 문제를 가지고 있었다. 

### 탐색을 하여 상위 노드 → 하위 노드로 진행해야한다.

루트를 기준으로 탐색을 통해 결정되어야 한다. 따라서 BFS 또는 DFS를 사용해 루트에서부터 아래 방향으로 탐색하며, 부모 노드를 기록해야 한다.

아래는 BFS를 이용한 부모 노드 설정 로직이다.

```java
static int[] solve(List<Integer>[] graph, int n) {
		Queue<Integer> queue = new ArrayDeque<>();
		int[] parent = new int[n + 1];
		queue.add(1);
		
		while (!queue.isEmpty()) {
			int node = queue.poll();
			for (int next_node : graph[node]) {
				if (parent[next_node] > 0) continue;
				parent[next_node] = node;
				queue.add(next_node);
			}
		}
		return parent;
	}
```

- 현재 탐색 중인 노드(node)는 이미 루트 기준으로 도달한 노드이므로 부모 노드가 확정된 상태이다.
- 이 노드에서 처음 방문하는 인접 노드(`nextNode`)는 자연스럽게 자식 노드가 된다.
- 따라서 이 시점에서 `parent[nextNode] = node`로 부모 관계를 설정하는 것이 논리적으로 안전하다.

### 코딩 테스트에선 StringBuilder를 사용하자


처음엔 아래와 같이 매번 출력했다.

```java
System.out.println(parent[i]);
```

채점 시간이 너무 오래 걸려서, `StringBuilder`를 사용해 한 번에 출력하도록 변경했다. 

```java
int[] parent = solve(graph, n);
for (int i = 2; i < n + 1; i++) {
		sb.append(parent[i]).append("\n");
}
		System.out.println(sb.toString());
```

이 방식으로 변경하자 실행 시간이 크게 감소하는 것을 확인 할 수 있었다.

## 최종 코드 (Java)

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.ArrayDeque;
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;
import java.util.Queue;
import java.util.StringTokenizer;

public class _11725 {

	public static void main(String[] args) throws IOException {

		BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
		StringBuilder sb = new StringBuilder();

		int n = Integer.parseInt(br.readLine());
		List<Integer>[] graph = new ArrayList[n + 1]; 
		for (int i = 1; i < n + 1; i++) {
			graph[i] = new ArrayList<>();
		}
		
		for (int i = 0; i < n - 1; i++) {
			StringTokenizer st = new StringTokenizer(br.readLine());
			int n1 = Integer.parseInt(st.nextToken());
			int n2 = Integer.parseInt(st.nextToken());
			graph[n1].add(n2);
			graph[n2].add(n1);
		}
		
		
		int[] parent = solve(graph, n);
		for (int i = 2; i < n + 1; i++) {
			sb.append(parent[i]).append("\n");
		}
		System.out.println(sb.toString());
	}
	
	static int[] solve(List<Integer>[] graph, int n) {
		Queue<Integer> queue = new ArrayDeque<>();
		int[] parent = new int[n + 1];
		queue.add(1);
		
		while (!queue.isEmpty()) {
			int node = queue.poll();
			for (int next_node : graph[node]) {
				if (parent[next_node] > 0) continue;
				parent[next_node] = node;
				queue.add(next_node);
			}
		}
		return parent;
	}
}

```
